# テキスト図・表 描画スキル（全角文字対応）

## 前提とルール

### 出力形式
- **必ずコードブロック（` ``` `）で囲む**
- コードブロック外では使わない
- 図の中に Markdown 装飾（`**`、`_` など）を混在させない

### 等幅フォント前提
- 半角文字（ASCII 英数字・記号）= 表示幅 **1**
- 全角文字（CJK・全角記号）= 表示幅 **2**
- すべての行の合計表示幅を揃える

---

## 文字セット（どちらか一方に統一し、混在させない）

### シンプル（推奨・再現性が高い）
```
角: +
横: -
縦: |
矢印: -> <- => |
空白: 半角スペース
```

**迷ったら常にこちらを使う。** ASCII のみで構成されるため、ほぼすべての環境でズレない。

### 拡張（見栄えは良いが注意が必要）
```
角: ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼
横: ─
縦: │
矢印: → ← ⇒ ↓ ↑
```

> ⚠️ **曖昧幅（East Asian Width = Ambiguous）の罠**
> `─` `│` `→` などの罫線・矢印は、環境によって表示幅が 1 にも 2 にもなる（ターミナル、Markdown ビューア、フォント設定に依存）。
> 再現性が必要な場面ではシンプルセットを使う。拡張セットを使う場合は、単一環境での表示のみを想定する。

### 使ってはいけないもの
- **絵文字**（`✅` `📝` など）: 環境依存で幅が 1〜2 のいずれにもなる
- **全角スペース（`　`）**: 不可視で混入に気づきにくい。半角スペースのみ使う
- **合字・結合文字**: 幅の扱いが環境依存

---

## 文字幅計算ルール

### 基本ルール
- 半角文字 = 幅 **1**
- 全角文字 = 幅 **2**
- 曖昧幅文字 = **使わない**（どうしても使う場合は環境を固定）

行の表示幅は、各文字の幅を 1 文字ずつ足し合わせた合計。

### スペース数の決め方（重要）

**目視で数えない。必ず計算する。**

```
末尾スペース数 = 目標内側幅 - コンテンツ表示幅 - 先頭スペース数
```

#### 例: 内側幅 27 のボックスに「フッター」を中央寄せ

1. 各文字を計算: フ(2)+ッ(2)+タ(2)+ー(2) = **8**
2. 先頭スペースを設計上決める: **8**
3. 末尾スペース = 27 - 8 - 8 = **11**
4. 結果: `|        フッター           |` (8 + 8 + 11 = 27) ✓

### 例: 全角を含む列幅計算

```
// NG（「名前」行の末尾が 1 文字ぶん右へずれる）
+----------+------+
| 名前      | 年齢  |
| Alice    | 25   |

// OK（全角 = 幅 2 として再計算）
+----------+------+
| 名前     | 年齢 |
| Alice    | 25   |
```

「名前」は全角 2 文字 → 表示幅 4。末尾スペース = セル内側幅(8) - 先頭(1) - 4 = **3**。

---

## 描画手順

1. **外枠を先に固定する**（幅と高さを決定）
2. **列幅を決める**（列ごとに最大表示幅を求め、全角を考慮）
3. **各行を描く前にスペース数を計算する**
   - コンテンツの各文字の幅を 1 文字ずつ足す
   - `末尾スペース = 目標内側幅 - コンテンツ幅 - 先頭スペース` で求める
   - 求めた数だけスペースを置く（目視調整は厳禁）
4. **チェックリストで確認する**（下記参照）

### 試行錯誤を最終出力に残さない

計算の途中でやり直しや修正が発生することは珍しくない。ただし**最終出力（Markdown ファイル、回答本文）には、確定した計算結果と図だけを残す**。

- 「だめ」「NG」「再設計」「列指定をやり直す」などの途中経過は、最終ドキュメントには含めない
- 計算ミスに気づいたら、該当箇所を書き直し、以前の誤った記述を残さない
- 「設計 → 検算 → 清書」のうち、ユーザーに渡すのは**清書だけ**
- 思考過程を見せたい場合でも、確定後に整理し直してから提示する

## よくあるズレのパターン

| パターン | 原因 | 対策 |
|---|---|---|
| 全角列が右にずれる | 全角を幅 1 で数えた | 全角 = 2 で再計算 |
| ヘッダーとデータ行がずれる | パディング量が違う | 全行の表示幅を揃える |
| 環境によって崩れる | 曖昧幅文字（`─` `│` `→` など）を使った | シンプルセット（`+` `-` `|` `->`）に差し替え |
| コピペ後に崩れる | 全角スペースや不可視文字の混入 | 半角スペースのみ使用 |
| 入れ子構造がずれる | 内枠を先に描いた | 外枠 → 内枠 の順に描く |
| スペース数が合わない | 目視で調整した | `末尾 = 目標 - コンテンツ - 先頭` で計算 |

## 出力前チェックリスト

- [ ] コードブロックで囲んだか？
- [ ] スペース数を `目標幅 - コンテンツ幅 - 先頭スペース` で計算したか（目視禁止）？
- [ ] 全角文字を 1 文字ずつ幅 2 として足し合わせたか？
- [ ] 文字セットが統一されているか（`+` と `┌` を混在させていないか）？
- [ ] 曖昧幅文字（`─` `│` `→` など）や絵文字を使っていないか？
- [ ] 全角スペースが混ざっていないか（半角スペースのみ使用）？
- [ ] 途中の試行錯誤やボツ設計が最終出力に残っていないか？

## 正しい例

### 全角文字を含む表

```
+----------+--------+----------+
| 項目     | 値     | 備考     |
+----------+--------+----------+
| 名前     | Alice  | 英語名   |
| 年齢     | 25     |          |
| 住所     | 東京   | 都内     |
+----------+--------+----------+
```

### 画面レイアウト

```
+---------------------------+
|    ヘッダー (Header)      |
+-----------+---------------+
| メニュー  | コンテンツ    |
| ・ホーム  |               |
| ・設定    |               |
+-----------+---------------+
|        フッター           |
+---------------------------+
```

### フロー図

```
+----------+     +----------+     +----------+
| 入力画面 | --> | 確認画面 | --> | 完了画面 |
+----------+     +----------+     +----------+
                      |
                      v
               +------------+
               | エラー表示 |
               +------------+
```
